# Автоматизированное тестирование с Cypress

Проект автоматизированного тестирования для сайта https://rahulshettyacademy.com/seleniumPractise/#/

## Описание проекта

Этот проект содержит автоматизированные end-to-end тесты, написанные на JavaScript с использованием фреймворка Cypress. Тесты проверяют функциональность корзины покупок на учебном сайте.

## Тестовые сценарии

Тесты выполняют следующие действия:

1. Переход на сайт https://rahulshettyacademy.com/seleniumPractise/#/
2. Ввод значения "ro" в строку поиска
3. Установка количества товара Carrot равным 5 через поле ввода (input)
4. Установка количества товара Mushroom равным 3 через кнопку увеличения (increment)
5. Добавление Carrot в корзину
6. Добавление Mushroom в корзину
7. Открытие корзины через иконку
8. Удаление Carrot из корзины
9. Проверка, что в корзине остался только Mushroom

## Технологии

- **JavaScript** - язык программирования
- **Cypress** - фреймворк для автоматизированного тестирования
- **Node.js** - среда выполнения

## Предварительные требования

Убедитесь, что на вашем компьютере установлены:

- Node.js (версия 18.x или выше)
- npm (обычно устанавливается вместе с Node.js)

## Установка

1. Клонируйте репозиторий:
```bash
git clone <URL-вашего-репозитория>
cd <название-папки-проекта>
```

2. Установите зависимости:
```bash
npm install
```

## Запуск тестов

### Headless режим (без графического интерфейса)
```bash
npm test
```

### Headed режим (с отображением браузера)
```bash
npm run test:headed
```

### Интерактивный режим (Cypress Test Runner)
```bash
npm run cypress:open
```

## Конфигурация

Основные настройки Cypress находятся в файле `cypress.config.js`:

- `baseUrl`: базовый URL тестируемого сайта
- `viewportWidth/Height`: размер окна браузера
- `video`: отключена запись видео
- `screenshotOnRunFailure`: скриншоты при ошибках

## Результаты тестирования

После выполнения тестов:

- Скриншоты при ошибках сохраняются в `cypress/screenshots/`
- Видео (если включено) сохраняются в `cypress/videos/`
